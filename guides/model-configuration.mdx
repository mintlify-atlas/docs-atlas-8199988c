## Understanding Thinking Modes

<Info>
Thinking modes enable Claude to show its reasoning process before providing a final answer, particularly useful for complex problem-solving.
</Info>

### Thinking Variants

Models with thinking capabilities support three budget levels:

| Variant | Thinking Budget | Use Case |
|---------|----------------|----------|
| `low` | 8,192 tokens | Simple reasoning tasks |
| `medium` | 16,384 tokens | Moderate complexity problems |
| `max` | 32,768 tokens | Complex, multi-step reasoning |

### Model Mappings

The plugin internally maps model names to AWS Kiro identifiers:

```typescript
const MODEL_MAPPING: Record<string, string> = {
  'claude-sonnet-4-5': 'CLAUDE_SONNET_4_5_20250929_V1_0',
  'claude-sonnet-4-6': 'claude-sonnet-4.6',

  'claude-haiku-4-5': 'CLAUDE_HAIKU_4_5_20251001_V1_0',
```

## Context Window Options

<Tip>
For large codebases or extensive documentation, use the 1M context variants to process more information in a single request.
</Tip>

### Standard Context (200K)

- **Best for**: Most coding tasks, code reviews, refactoring
- **Models**: All standard model variants
- **Token limit**: 200,000 input tokens

### Extended Context (1M)

- **Best for**: Entire repository analysis, large documentation sets
- **Models**: Sonnet 4.5/4.6 1M, Opus 4.6 1M variants
- **Token limit**: 1,000,000 input tokens

## Input Modalities

Different models support different input types:

### Text + Image + PDF
- Claude Sonnet 4.5/4.6 (all variants)
- Claude Opus 4.5/4.6 (all variants)

### Text + Image
- Claude Haiku 4.5

### Text Only
- Qwen3 Coder 480B

<Note>
All models support a maximum output of 64,000 tokens regardless of their context window size.
</Note>

## Choosing the Right Model

<Steps>
  <Step title="Start with Haiku for speed">
    Use `claude-haiku-4-5` for quick tasks like code completion or simple refactoring where speed matters most.
  </Step>
  
  <Step title="Use Sonnet for balance">
    Choose `claude-sonnet-4-6` for most development tasks requiring high-quality responses with reasonable speed.
  </Step>
  
  <Step title="Reserve Opus for complexity">
    Select `claude-opus-4-6` when you need the highest quality for complex architectural decisions or challenging debugging.
  </Step>
  
  <Step title="Enable thinking for reasoning">
    Add the `-thinking` suffix when you want to see the model's reasoning process, especially for complex problem-solving.
  </Step>
</Steps>
