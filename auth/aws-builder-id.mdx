AWS Builder ID is the default authentication method for the OpenCode Kiro Auth plugin. It provides free access to Claude models through AWS CodeWhisperer/Q Developer with no additional configuration required.

## Overview

AWS Builder ID uses OAuth 2.0 device authorization flow to authenticate users. The plugin handles the entire OAuth flow automatically, including:

- Client registration with AWS SSO OIDC
- Device code authorization
- Token polling and refresh
- Automatic token renewal

## Authentication Flow

<Steps>
  <Step title="Start Authentication">
    Run the authentication command:
    
    ```bash
    opencode auth login
    ```
    
    Select **Other**, type `kiro`, and press enter.
  </Step>
  
  <Step title="Skip IAM Identity Center">
    When prompted for "IAM Identity Center Start URL", leave it **blank** and press enter.
    
    When prompted for "IAM Identity Center region", leave it **blank** and press enter.
    
    <Info>
    Leaving both prompts blank automatically defaults to AWS Builder ID authentication.
    </Info>
  </Step>
  
  <Step title="Complete Browser Authorization">
    A browser window will open automatically to the AWS verification page:
    
    ```
    https://view.awsapps.com/start/#/device?user_code=XXXX-XXXX
    ```
    
    If the browser doesn't open automatically, copy the URL and code from your terminal.
  </Step>
  
  <Step title="Sign In">
    On the AWS verification page:
    
    1. Enter the verification code displayed in your terminal
    2. Sign in with your AWS Builder ID (or create a new account)
    3. Approve the authorization request
  </Step>
  
  <Step title="Verification Complete">
    Once authorized, OpenCode will automatically:
    
    - Poll for the access token
    - Store credentials in `~/.config/opencode/kiro.db`
    - Fetch your usage limits
    - Make the account available for API requests
  </Step>
</Steps>

## OAuth Implementation Details

### Client Registration

The plugin registers an OIDC client with AWS SSO during authentication:

```typescript
// From: src/kiro/oauth-idc.ts:36-64
const registerResponse = await fetch(`${ssoOIDCEndpoint}/client/register`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'User-Agent': 'KiroIDE'
  },
  body: JSON.stringify({
    clientName: 'Kiro IDE',
    clientType: 'public',
    scopes: [
      'codewhisperer:completions',
      'codewhisperer:analysis',
      'codewhisperer:conversations',
      'codewhisperer:transformations',
      'codewhisperer:taskassist'
    ],
    grantTypes: ['urn:ietf:params:oauth:grant-type:device_code', 'refresh_token']
  })
})

const { clientId, clientSecret } = await registerResponse.json()
```

### Device Authorization

After client registration, the plugin initiates device authorization:

```typescript
// From: src/kiro/oauth-idc.ts:66-86
const deviceAuthResponse = await fetch(`${ssoOIDCEndpoint}/device_authorization`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'User-Agent': 'KiroIDE'
  },
  body: JSON.stringify({
    clientId,
    clientSecret,
    startUrl: 'https://view.awsapps.com/start'
  })
})

const {
  verificationUri,
  verificationUriComplete,
  userCode,
  deviceCode,
  interval = 5,
  expiresIn = 600
} = await deviceAuthResponse.json()
```

### Token Polling

The plugin polls the token endpoint until the user completes authorization:

```typescript
// From: src/kiro/oauth-idc.ts:119-255
export async function pollKiroIDCToken(
  clientId: string,
  clientSecret: string,
  deviceCode: string,
  interval: number,
  expiresIn: number,
  region: KiroRegion
): Promise<KiroIDCTokenResult> {
  const maxAttempts = Math.floor(expiresIn / interval)
  let currentInterval = interval * 1000
  let attempts = 0

  while (attempts < maxAttempts) {
    attempts++
    await new Promise((resolve) => setTimeout(resolve, currentInterval))

    const tokenResponse = await fetch(`${ssoOIDCEndpoint}/token`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'User-Agent': 'KiroIDE'
      },
      body: JSON.stringify({
        clientId,
        clientSecret,
        deviceCode,
        grantType: 'urn:ietf:params:oauth:grant-type:device_code'
      })
    })

    const tokenData = await tokenResponse.json()

    if (tokenData.error === 'authorization_pending') {
      continue // User hasn't authorized yet
    }

    if (tokenData.error === 'slow_down') {
      currentInterval += 5000 // Increase polling interval
      continue
    }

    if (tokenData.access_token && tokenData.refresh_token) {
      return {
        accessToken: tokenData.access_token,
        refreshToken: tokenData.refresh_token,
        expiresAt: Date.now() + (tokenData.expires_in || 3600) * 1000,
        clientId,
        clientSecret,
        region,
        authMethod: 'idc'
      }
    }
  }

  throw new Error('Token polling timed out')
}
```

## Token Management

### Access Token Expiry

The plugin automatically refreshes access tokens before they expire:

```typescript
// From: src/kiro/auth.ts:14-17
export function accessTokenExpired(auth: KiroAuthDetails, bufferMs = 120000): boolean {
  if (!auth.access || !auth.expires) return true
  return Date.now() >= auth.expires - bufferMs
}
```

<Note>
By default, tokens are refreshed 2 minutes (120,000ms) before expiry. Configure this with `token_expiry_buffer_ms` in `~/.config/opencode/kiro.json`.
</Note>

### Token Refresh Flow

When tokens expire, the plugin automatically refreshes them:

```typescript
// From: src/core/auth/token-refresher.ts:25-42
async refreshIfNeeded(
  account: ManagedAccount,
  auth: KiroAuthDetails,
  showToast: ToastFunction
): Promise<{ account: ManagedAccount; shouldContinue: boolean }> {
  if (!accessTokenExpired(auth, this.config.token_expiry_buffer_ms)) {
    return { account, shouldContinue: false }
  }

  try {
    const newAuth = await refreshAccessToken(auth)
    this.accountManager.updateFromAuth(account, newAuth)
    await this.repository.batchSave(this.accountManager.getAccounts())
    return { account, shouldContinue: false }
  } catch (e: any) {
    return await this.handleRefreshError(e, account, showToast)
  }
}
```

## Storage

Authentication credentials are stored in a SQLite database:

**Linux/macOS:**
- Database: `~/.config/opencode/kiro.db`
- Config: `~/.config/opencode/kiro.json`

**Windows:**
- Database: `%APPDATA%\opencode\kiro.db`
- Config: `%APPDATA%\opencode\kiro.json`

### Account Schema

Accounts are stored with the following structure:

```typescript
// From: src/plugin/accounts.ts:38-59
interface ManagedAccount {
  id: string                    // Deterministic hash of email + method + clientId + profileArn
  email: string                 // User's email from AWS
  authMethod: 'idc' | 'desktop' // Authentication method
  region: KiroRegion            // AWS service region (us-east-1, us-west-2)
  oidcRegion?: KiroRegion       // OIDC region for SSO
  clientId?: string             // OAuth client ID
  clientSecret?: string         // OAuth client secret
  profileArn?: string           // AWS Q Developer profile ARN (for IAM Identity Center)
  startUrl?: string             // IAM Identity Center start URL
  refreshToken: string          // OAuth refresh token
  accessToken: string           // OAuth access token
  expiresAt: number             // Token expiration timestamp
  rateLimitResetTime: number    // Rate limit reset timestamp
  isHealthy: boolean            // Account health status
  unhealthyReason?: string      // Reason if unhealthy
  recoveryTime?: number         // Timestamp when account can be retried
  failCount: number             // Number of consecutive failures
  lastUsed?: number             // Last usage timestamp
  usedCount: number             // Total API requests made
  limitCount: number            // Daily usage quota
}
```

## Deterministic Account IDs

The plugin generates deterministic account IDs to prevent duplicates:

```typescript
// From: src/plugin/accounts.ts:14-23
export function createDeterministicAccountId(
  email: string,
  method: string,
  clientId?: string,
  profileArn?: string
): string {
  return createHash('sha256')
    .update(`${email}:${method}:${clientId || ''}:${profileArn || ''}`)
    .digest('hex')
}
```

<Info>
Account IDs are SHA-256 hashes that ensure the same credentials always map to the same account record.
</Info>

## Configuration

Customize AWS Builder ID authentication in `~/.config/opencode/kiro.json`:

```json
{
  "default_region": "us-east-1",
  "token_expiry_buffer_ms": 120000,
  "usage_tracking_enabled": true,
  "auto_sync_kiro_cli": true
}
```

### Configuration Options

- **`default_region`**: AWS service region (`us-east-1` or `us-west-2`)
- **`token_expiry_buffer_ms`**: Token refresh buffer time (default: 120000ms = 2 minutes)
- **`usage_tracking_enabled`**: Enable usage tracking and toast notifications
- **`auto_sync_kiro_cli`**: Auto-import credentials from Kiro CLI

## Troubleshooting

<Accordion title="Browser doesn't open automatically">
  If the browser doesn't open:
  
  1. Copy the verification URL from your terminal
  2. Paste it into your browser
  3. Enter the verification code shown in the terminal
</Accordion>

<Accordion title="Token refresh fails">
  If token refresh fails:
  
  ```bash
  # Re-authenticate
  opencode auth login
  ```
  
  The plugin will automatically mark the old account as unhealthy and create a new one.
</Accordion>

<Accordion title="No accounts error">
  If you see "No accounts" error:
  
  1. Ensure authentication completed successfully
  2. Check that `~/.config/opencode/kiro.db` exists
  3. Enable auto-sync in config: `"auto_sync_kiro_cli": true`
  4. Re-run authentication: `opencode auth login`
</Accordion>
